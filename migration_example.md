# Example: Adding a New Feature with Alembic

## Scenario: Add User Preferences Table

### 1. Update Your SQLAlchemy Model

Add new model to `src/infrastructure/database/models.py`:

```python
class UserPreferencesModel(Base):
    """User preferences and settings"""
    __tablename__ = 'user_preferences'
    
    id = Column(UUID(as_uuid=True), primary_key=True, default=uuid4)
    user_id = Column(UUID(as_uuid=True), ForeignKey('users.id'), nullable=False)
    theme = Column(String(20), default='light')
    notifications_enabled = Column(Boolean, default=True)
    preferred_agent_type = Column(String(50), default='therapy')
    language = Column(String(10), default='en')
    timezone = Column(String(50), default='UTC')
    
    created_at = Column(DateTime(timezone=True), server_default=func.now())
    updated_at = Column(DateTime(timezone=True), server_default=func.now(), onupdate=func.now())
    
    # Relationship
    user = relationship("UserModel", backref="preferences")
```

### 2. Generate Migration

```bash
# Alembic detects your model changes and creates migration
python3 manage_db.py new-migration --message "Add user preferences table"

# This creates: migrations/versions/2024_01_15_1234-add_user_preferences_table.py
```

### 4. Apply Migration to Database

```bash
# Apply to development
python3 manage_db.py migrate

# Apply to production (same command, different environment)

```

### 5. If Something Goes Wrong - Rollback

```bash
# Undo last migration
python3 manage_db.py rollback

# Or undo multiple steps
python3 manage_db.py rollback --steps 3
```

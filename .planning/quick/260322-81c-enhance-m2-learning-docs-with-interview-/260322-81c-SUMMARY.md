## What Was Added to Each File

### prometheus_fastapi.md

- **When to use Prometheus:** 7-row comparison table covering pull vs push, sub-second resolution, multi-process production, vs OTel traces
- **Advanced code examples:** Summary vs Histogram decision rationale; reading metric values in tests via `_value.get()`; programmatic `/metrics` assertion mirroring `demo_steps/10_metrics.sh`; Alertmanager rule for crisis detection spike
- **Interview Prep:** 10 Q&A pairs — Counter/Gauge/Histogram distinctions, custom bucket rationale, high-cardinality dangers, `rate()` vs `irate()`, lifespan startup ordering, how the instrumentator ASGI middleware works, counter reset on restart, multiprocess mode, `should_group_status_codes`, Pushgateway use case
- **Gotchas:** Duplicate metric registration error, `le` label meaning, `histogram_quantile` accuracy limits, scrape interval blind spots, instrumentation order bug, `track_inprogress()` on exception

### celery_redis.md

- **When to use Celery:** 7-row comparison table covering Celery vs FastAPI BackgroundTasks, durability needs, Celery Beat, canvas fan-out, Redis Pub/Sub
- **Advanced code examples:** Canvas chain pattern with `notify_new_record` and `generate_weekly_summary`; full `@shared_task` retry config with comments explaining each flag; Flower polling pattern in bash and Python; `task.apply()` for broker-free unit tests
- **Interview Prep:** 10 Q&A pairs — broker/backend roles, `@shared_task` vs `@celery_app.task`, worker import isolation, post-commit dispatch, `acks_late=True`, `autoretry_for` vs manual retry, Flower capabilities, `delay()` vs `apply_async()`, `async def` task body danger, idempotency design
- **Gotchas:** Pre-commit dispatch, beat vs worker process distinction, concurrent task execution without `acks_late`, `bind=True` purpose, swallowed enqueue exceptions and observability gap

### opentelemetry.md

- **When to use OTel:** 8-row comparison table covering traces vs metrics vs logs per diagnostic question, distributed tracing, AsyncPGInstrumentor vs logging, OTel alerting limitation
- **Advanced code examples:** Error status recording with `span.record_exception()`; span events as lightweight breadcrumbs; cross-process Celery trace propagation with `inject()`/`extract()`; integration test pattern with `InMemorySpanExporter` and `StatusCode` assertion
- **Interview Prep:** 10 Q&A pairs — three observability pillars, trace vs span, decorator form async bug, asyncio context propagation, `AsyncPGInstrumentor` ordering, `BatchSpanProcessor` vs `SimpleSpanProcessor` in tests, `get_tracer(__name__)` scope, W3C Trace Context, OTLP double-path gotcha, automatic parent-child span nesting
- **Gotchas:** Decorator form 0ms spans visible in Jaeger, global `TracerProvider` singleton, silent span drop on Jaeger outage, `InMemorySpanExporter.clear()` requirement, `AsyncPGInstrumentor` vs `SQLAlchemyInstrumentor` distinction

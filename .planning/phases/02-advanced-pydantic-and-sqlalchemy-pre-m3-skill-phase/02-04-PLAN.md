**Title:** `# SQLAlchemy 2.0 Advanced Patterns — EmotionAI study guide`

**Section 1: "## What is it and why do we use it here"**
- Explain SQLAlchemy 2.0's async-first design and the ORM patterns that prevent common async bugs
- Explain why EmotionAI needs these patterns:
  - DetachedInstanceError is the #1 async SQLAlchemy bug — lazy-loading outside sessions crashes
  - hybrid_property avoids adding DB columns for derived values
  - selectinload prevents N+1 queries (e.g., loading user conversations with messages)
  - Consistent session management (get_session context manager) ensures commit/rollback safety
- Reference: this project uses SQLAlchemy 2.0.48 with asyncpg

**Section 2: "## How it works conceptually (explain as if to a junior developer)"**
- **Relationship loading strategies:** Explain lazy (default, dangerous in async), selectinload (separate IN query, good for collections), joinedload (SQL JOIN, good for scalars), subqueryload (subquery, less common). Use diagrams or examples showing the SQL each generates.
- **hybrid_property:** Explain dual-nature — Python property on instances, SQL expression in queries. Show how the `@expression` decorator enables WHERE clause usage.
- **Async session lifecycle:** Explain the session context manager pattern, why `expire_on_commit=False` matters, and the difference between `get_session()` (managed) vs `async_session_factory()` (unmanaged).
- **select() API:** Explain the 2.0-style `select(Model).where(...)` vs legacy `session.query(Model).filter(...)` and why the project uses the modern API.

**Section 3: "## Key patterns used in this project (with code examples from the actual codebase)"**
Include actual code from the modified files. Cover:

1. **hybrid_property: intensity_level** — show the EmotionalRecordModel.intensity_level property from `models.py`. Show both the Python-level and SQL expression sides. Explain how `case()` maps to SQL CASE WHEN.
2. **hybrid_property: duration_days** — show ConversationModel.duration_days from `models.py`. Explain why this one only has a Python-level implementation (runtime-dependent value).
3. **selectinload usage** — show `get_conversation_with_messages` from the conversation repository. Explain the `.options(selectinload(ConversationModel.messages))` pattern and what SQL it generates (two SELECT statements).
4. **selectinload for user context** — show `get_by_user_id_with_user` from the emotional record repository. Explain when you need this (M3 personalization context building).
5. **Session management: get_session()** — show the before/after of the conversation repository migration from `async_session_factory()` to `get_session()`. Explain the safety benefits.
6. **Relationship definitions** — show the UserModel relationships from `models.py` (conversations, emotional_records, etc.) and explain `back_populates`, `cascade`, `uselist=False`.

**Section 4: "## Common mistakes and how to avoid them"**
Cover at least 5 mistakes:
1. Accessing lazy-loaded relationship outside async session (DetachedInstanceError)
2. Using joinedload for one-to-many collections (cartesian product)
3. Using async_session_factory() directly instead of get_session() (no automatic rollback)
4. Forgetting `expire_on_commit=False` in session maker (expired attributes post-commit)
5. Using legacy session.query() API instead of select() (not async-compatible)
6. Defining hybrid_property SQL expression that doesn't match Python logic (inconsistent results)

**Section 5: "## Further reading"**
- SQLAlchemy 2.0 relationship loading: https://docs.sqlalchemy.org/en/20/orm/queryguide/relationships.html
- SQLAlchemy hybrid properties: https://docs.sqlalchemy.org/en/20/orm/extensions/hybrid.html
- SQLAlchemy async session: https://docs.sqlalchemy.org/en/20/orm/extensions/asyncio.html
- SQLAlchemy 2.0 migration guide: https://docs.sqlalchemy.org/en/20/changelog/migration_20.html
- asyncpg + SQLAlchemy best practices: https://docs.sqlalchemy.org/en/20/dialects/postgresql.html#module-sqlalchemy.dialects.postgresql.asyncpg
